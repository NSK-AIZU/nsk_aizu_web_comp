CSSフレームワークは使わず、単一のHTMLファイル内にVanilla CSS/JSで記述してください。

# 仕様書

## 1. 目的
ユーザーの画面サイズを「田んぼ」と仮定し、1シーズン（年1回）で作れるご飯の量（g）を計算する。
デバイス画面がいかに小さいか（お米を作るには広い土地が必要か）を実感してもらう。

## 2. 計算ロジック
### A. 物理サイズの取得（重要：補正あり）
現代の高精細ディスプレイに対応するため、実測データに基づく補正係数を適用します。

1. モバイル判定: UserAgentでスマホ/タブレットを検知。
2. 補正係数 (ScaleFactor):
   - モバイルの場合: 0.64
   - PCの場合: 0.75
3. 長さ(cm) = (ピクセル数 / 96 * 2.54) * ScaleFactor

### B. 収穫量の計算
- 10アール(1000m^2)あたりの玄米収穫量: 540kg
- 精米歩留まり: 0.9
- 炊飯倍率: 2.3
- 計算式: ご飯(g) = 面積(m^2) * 540 * 0.9 * 2.3

## 3. 表示機能・UI
### A. 結果表示
- 「画面サイズ (cm x cm)」を表示（「※実寸補正済み」と注釈）。
- 「収穫できるご飯の量 (g)」をメインで大きく表示（小数点第2位まで）。

### B. メッセージ分岐
- 基準値: お茶碗一杯 = 150g
- 【ケース1：150g未満（不足）】
  - 「お茶碗一杯には足りません！」と強調（赤/オレンジ系文字）。
  - 「一杯分にするには、この端末があと【 X 台 】必要です」と計算して表示。
- 【ケース2：150g以上（充足）】
  - 「すごい！この画面サイズなら自給自足可能です」と表示（緑系文字）。
  - 「お茶碗 約【 Y 杯 】分になります」と表示。

### C. 可視化
- 米粒のような見た目のdiv要素を、計算された粒数分（上限50個程度）描画して視覚的に見せる。

## 4. デザイン要件
- 清潔感のある「農業・自然」をイメージした緑ベースのデザイン。
- カード型レイアウトで、中央揃え。
- スマホでも見やすいレスポンシブ対応。
- CSSは<style>タグ内に記述すること。`
