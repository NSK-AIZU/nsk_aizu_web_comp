## A: データ活用

### 問1

#### 問題概要
レシートのデータから、時間帯ごとの売上分析と曜日別の商品購買状況分析に必要な情報を選択する問題

#### 解答
- **ア**: 5 (購入した商品の合計金額)
- **イ**: 3 (商品コード、購入商品名)
- **ウ**: 4 (購入した商品の個数)

#### 解説

##### 時間帯ごとの総売上額の比較に必要な情報（ア）
LikeWing全体での「時間帯ごとの総売上額（消費税込）」を比較するには、図1の「購入時刻」と組み合わせる情報が必要です。

売上額を集計するためには、**⑤購入した商品の合計金額**が必要です。これにより、各購入時刻における売上額を集計し、時間帯別に分析できます。

##### 曜日別の各商品の購買状況の把握に必要な情報（イ・ウ）
「曜日別の各商品の購買の状況」を把握するには、図1の「購入日、曜日」と組み合わせる情報が必要です。

- **どの商品が購入されたか**を知るには：**③商品コード、購入商品名**
- **何個購入されたか**を知るには：**④購入した商品の個数**

この2つの情報により、「○曜日に△△という商品が□個購入された」という分析が可能になります。

#### ポイント
- **売上分析**：金額データ（合計金額）が必要
- **購買状況分析**：商品の識別情報（商品コード・名称）と数量データが必要
- レシートには様々な情報があるが、分析目的に応じて必要な情報を選択することが重要

### 問2

#### 問題概要
下線部Aの分析（ポイント会員情報とレシートの組み合わせ分析）によって得られない情報を選ぶ問題

#### 解答
- **エ**: 0 (顧客が商品を購入した理由)

#### 解説

##### 利用可能な情報
下線部Aでは、以下の2つの情報源を組み合わせて分析します：
- **ポイント会員情報**：名前、性別、生年月日（3つの属性情報）
- **レシートの情報**：購入時刻、商品コード、購入商品名、個数、金額など

##### 各選択肢の検討

**⓪ 顧客が商品を購入した理由**
- これはレシートにもポイント会員情報にも記録されていない
- 購入理由を知るにはアンケートやインタビューなど別の手段が必要
- **得られない情報** ✓

**① 同じ顧客に、繰り返し購入される傾向がある商品**
- 会員IDと購入履歴を紐付けることで、同一顧客の複数回の購入を追跡可能
- リピート購入の分析が可能
- 得られる情報

**② ある商品を多く購入している顧客の年齢層**
- 生年月日から年齢が計算できる
- 商品の購入データと組み合わせることで分析可能
- 得られる情報

**③ 年齢や性別の違いによる、来店する時間帯の傾向**
- 性別と生年月日（→年齢）の情報がある
- 購入時刻の情報がある
- これらを組み合わせて分析可能
- 得られる情報

#### ポイント
- **定量データ**（いつ、何を、いくつ、いくら）は記録から得られる
- **定性データ**（なぜ、どう思ったか）は記録からは得られず、別途調査が必要
- データ分析の限界を理解することが重要

### 問3

#### 問題概要
LikeWingの情報システムにおける情報の流れで、店コードとポイント会員IDが必要とされる流れを選択する問題

#### 解答
- **オ**: 3 (あ、い)
- **カ**: 5 (い、う)

#### 解説

##### 情報の流れの確認
図2に示される情報の流れ：
- **(あ)**: LikeWing配送センター → LikeWing本部（配送情報など）
- **(い)**: LikeWing店舗 → LikeWing本部（売上・購買情報など）
- **(う)**: 顧客 → LikeWing店舗（商品購入）

##### オ：店コードが必要な情報の流れ

**店コードの役割**：店舗を識別するための情報

- **(あ) 配送センター → 本部**：配送情報に店コードが含まれる（どの店舗に配送したか）
- **(い) 店舗 → 本部**：売上・購買情報に店コードが必要（どの店舗の売上か）
- **(う) 顧客 → 店舗**：顧客が商品を購入する際、顧客自身が店コードという情報を持っている必要はない

**答え：あ、い（選択肢③）**

注：当初「う」も含まれると考えがちですが、情報の「流れ」として考えると、顧客から店舗への流れでは、店コードは顧客が提供する情報ではありません。店舗側が記録する情報です。

##### カ：ポイント会員IDが必要な情報の流れ

**ポイント会員IDの役割**：顧客を識別するための情報

- **(あ) 配送センター → 本部**：配送情報には会員IDは不要（店舗への配送なので）
- **(い) 店舗 → 本部**：購買情報として会員IDを送信（誰が購入したか）
- **(う) 顧客 → 店舗**：顧客がポイントカードを提示（会員IDを店舗に伝える）

**答え：い、う（選択肢⑤）**

#### ポイント
- **店コード**は店舗を識別する情報 → 店舗が関わる業務の流れで必要
- **会員ID**は顧客を識別する情報 → 顧客が関わる購買の流れで必要
- 情報システムでは、各情報がどの流れで必要かを適切に設計する必要がある

### 問4

#### 問題概要
下線部Cの運用（ポイント会員IDとネットショッピングアカウントの連携）によるメリットとその実現に必要な条件を選択する問題

#### 解答
- **キ**: 0 (あ)
- **ク**: 6 (あ、い、う)
- **ケ**: 3 (あ、い)

#### 解説

##### 想定するメリットと必要な条件

**メリットI：ポイント数の自動表示**
> 顧客がネットショッピングサイトにログインしたときに、現在のポイントカードのポイント数を自己に送り状態確認やチャンネルの自動的に表示される。

**必要な条件**：
- ポイント会員IDとネットショッピングのアカウントが紐付けされている（条件あ）

ポイント数は実店舗での購入でも貯まるため、ネットショッピングアカウントと実店舗のポイントカードが連携していないと、ポイント数を表示できません。

**答え：あ（キ = 0）**

---

**メリットII：在庫がある店舗情報の表示**
> 顧客がネットショッピングで商品を購入しようとするとき、その顧客がポイントカードを多く利用する店舗ゆうちょ、その商品の在庫がある実店舗の情報が表示される。

**必要な条件**：
- **あ**：どの顧客かを識別（ポイントカードの利用履歴を参照）
- **い**：ネットショッピングの商品と実店舗の商品を紐付け（商品に店舗番号が割り当てられている）
- **う**：各店舗の在庫数を確認（商品コードと店舗コードから在庫数を調べる）

3つの条件すべてが必要です。

**答え：あ、い、う（ク = 6）**

---

**メリットIII：おすすめ商品の提案**
> 顧客がネットショッピングサイトにログインしたときに、商品の購入履歴的な実績をもとに類似している他の顧客の購入履歴をもとに、おすすめ商品を提案に表示する。

**必要な条件**：
- **あ**：ポイント会員IDとアカウントの紐付け（実店舗での購入履歴を参照するため）
- **い**：商品に店舗番号が割り当てられている（実店舗とネットショップの商品を統合管理）

実店舗での購入履歴も含めて「類似した顧客」を見つけるため、実店舗とネットショップのデータ統合が必要です。

**答え：あ、い（ケ = 3）**

注：条件うは在庫数の確認に関するもので、おすすめ商品の提案には直接必要ありません。

#### ポイント
- **データ連携**の重要性：異なるシステム（実店舗とネットショップ）のデータを連携することで、より高度なサービスが実現できる
- **必要最小限の条件**を見極める：各機能に本当に必要な条件だけを選択する
- **プライバシー配慮**：会員IDとアカウントの紐付けは便利だが、適切な同意と管理が必要

## B: シミュレーション

### 問1 (B)

#### 問題概要
集金シミュレーションにおける千円札と一万円札の枚数変化を表に記入する問題

#### 解答
- **コ**: 5 (枚)
- **サ・シ**: 1, 2 (枚)
- **ス・セ**: 1, 8 (枚)

#### 解説

##### シミュレーションのルール
- メンバー10人から順に集金（6,000円ずつ）
- 支払い方法は乱数rで決定：
  - r ≤ 3: 千円札6枚で支払う
  - r ≥ 4: 一万円札1枚で支払う（おつり4,000円は千円札4枚）
- 千円札で支払う確率30%、一万円札で支払う確率70%

##### 表の計算

**初期状態**：
- 一万円札：0枚
- 千円札：0枚

**1人目（r=8）**：一万円札で支払う
- 一万円札を1枚受け取る → 1枚
- おつり4,000円（千円札4枚）を渡す → -4枚
- 手元：一万円札1枚、千円札-4枚

**2人目（r=1）**：千円札で支払う
- 千円札6枚を受け取る → -4+6=2枚
- 手元：一万円札1枚、千円札2枚

**3人目（r=6）**：一万円札で支払う
- 一万円札を1枚受け取る → 2枚
- おつり千円札4枚を渡す → 2-4=-2枚
- 手元：一万円札2枚、千円札-2枚

**4人目（r=10）**：一万円札で支払う
- 一万円札を1枚受け取る → 3枚
- おつり千円札4枚を渡す → -2-4=-6枚
- 手元：一万円札3枚、千円札-6枚

**5人目（r=9）**：一万円札で支払う
- 一万円札を1枚受け取る → 4枚
- おつり千円札4枚を渡す → -6-4=-10枚
- 手元：一万円札4枚、千円札-10枚

**6人目（r=4）**：一万円札で支払う
- 一万円札を1枚受け取る → 5枚
- おつり千円札4枚を渡す → -10-4=-14枚
- 手元：一万円札**5枚**、千円札-14枚
- **コ = 5**

**7人目（r=5）**：一万円札で支払う
- 一万円札を1枚受け取る → 6枚
- おつり千円札4枚を渡す → -14-4=-18枚
- 手元：一万円札6枚、千円札-18枚

**8人目（r=3）**：千円札で支払う
- 千円札6枚を受け取る → -18+6=-12枚
- 手元：一万円札6枚、千円札-12枚
- 千円札の枚数の変化：-18 → -12 なので、**-1**, **2**（増加）
- **サ = 1, シ = 2**

**9人目（r=7）**：一万円札で支払う
- 一万円札を1枚受け取る → 7枚
- おつり千円札4枚を渡す → -12-4=-16枚
- 手元：一万円札7枚、千円札-16枚

**10人目（r=2）**：千円札で支払う
- 千円札6枚を受け取る → -16+6=-10枚
- 手元：一万円札7枚、千円札-10枚

##### 事前準備の千円札枚数

問題文：「今回行った1回のシミュレーションの場合、千円札【ス】【セ】枚を事前に準備しておけば、一度も千円札が不足することなく集金できる」

シミュレーション中の千円札の最小値は**-18枚**（7人目終了時点）でした。

つまり、事前に**18枚**の千円札を準備しておけば、最も不足する時点でも18-18=0枚となり、マイナスになることはありません。

**ス・セ = 1, 8**

#### ポイント
- **シミュレーション**：ランダムな事象を再現して分析する手法
- **マイナスの在庫**：後で補填されることを前提とした記録方法
- **最小値の追跡**：最も厳しい状況を把握することでリスク管理が可能

### 問2 (B)

#### 問題概要
10,000回のシミュレーション結果（図3）から読み取れる考察を選択する問題

#### 解答
- **ソ**: 1 (最後まで千円札が不足しなかったのは、全回数の1割以下である)

#### 解説

##### 図3の読み取り
図3は、10,000回のシミュレーションにおける「手元の千円札の枚数」の最小値の分布を示すヒストグラムです。

- 横軸：手元の千円札の枚数の最小値（-40 〜 0）
- 縦軸：回数
- 最小値が0 = 最後まで千円札が不足しなかった

##### 各選択肢の検討

**⓪ 全員が一万円札で支払うケースはなかった**
- 全員が一万円札で支払った場合、千円札の最小値は0-(10人×4枚)=-40枚
- 図3では-40付近にもデータが存在する
- したがって、全員が一万円札で支払うケースもあった可能性がある
- **誤り**

**① 最後まで千円札が不足しなかったのは、全回数の1割以下である**
- 「最後まで不足しなかった」= 最小値が0以上
- 図3で最小値=0の回数は約800回
- 800/10,000 = 8% < 10%（1割）
- **正しい** ✓

**② 別の乱数を使って10,000回シミュレーションを行っても、最終的な結果のグラフはまったく同じになる**
- シミュレーションは乱数を使用するため、乱数列が変われば結果も変わる
- 分布の傾向は似るかもしれないが、「まったく同じ」にはならない
- **誤り**

**③ 全員が千円札でお金を支払ったケースが1回以上ある**
- 全員が千円札で支払った場合、千円札は常に増加するため最小値は0のまま
- しかし、最小値=0のケースは「全員が千円札」とは限らない
  - 例：前半に一万円札、後半に千円札という組み合わせでも最小値=0になり得る
- 図3からだけでは断定できない
- **不明（断定できない）**

#### ポイント
- **ヒストグラムの読み取り**：横軸と縦軸の意味を正確に理解する
- **確率的思考**：シミュレーションは確率的な現象なので、結果は毎回変わる
- **データの限界**：グラフから読み取れることと読み取れないことを区別する

### 問3 (B)

#### 問題概要
事前に千円札を20枚用意した場合に起こることがないケースを選択する問題

#### 解答
- **タ**: 2 (千円札で支払った人が5人いて、途中でおつりの千円札が不足するケース)

#### 解説

##### 初期条件
- 千円札：20枚
- メンバー：10人
- 各人の支払い：千円札6枚 or 一万円札1枚（おつり千円札4枚）

##### 各選択肢の検討

**⓪ 最初の1人が千円札で支払ったとしても、途中でおつりの千円札が不足するケース**

最悪のシナリオを考える：
- 1人目：千円札で支払う → 20+6=26枚
- 2〜10人目：全員が一万円札で支払う → 26-(9×4)=26-36=-10枚

途中で不足します。**起こり得る** ✗

**① 用意された千円札をまったく使うことなく全員からの集金を終えるケース**

全員が千円札6枚で支払えば：
- 20+(10×6)=80枚

千円札をまったく使わず（渡さず）に集金を終えられます。**起こり得る** ✗

**② 千円札で支払った人が5人いて、途中でおつりの千円札が不足するケース**

千円札で支払う人が5人の場合：
- 千円札の収入：5×6=30枚
- 一万円札で支払う人：5人
- 千円札の支出：5×4=20枚
- 合計：20+30-20=30枚（最終的に30枚残る）

順序による途中の不足を考える：

**最悪の順序**：最初の5人が全員一万円札で支払う場合
- 20-(5×4)=20-20=0枚（ギリギリ）

この後、残り5人が千円札で支払えば30枚になります。

つまり、千円札で支払う人が5人いる場合、初期の20枚があれば、どんな順序でも途中で不足することはありません。**起こらない** ✓

**③ 一万円札で支払った人が8人いて、途中でおつりの千円札が不足せず全員からの集金を終えるケース**

一万円札で支払う人が8人の場合：
- 千円札で支払う人：2人
- 千円札の収入：2×6=12枚
- 千円札の支出：8×4=32枚

**適切な順序**：千円札2人 → 一万円札8人
- 20+12-32=0枚（ギリギリOK）

この順序なら不足せずに終えられます。**起こり得る** ✗

#### 答えの理由

千円札で支払う人が5人いる場合、初期20枚があれば：
- 最悪でも 20-(5×4)=0枚まで
- その後の千円札での支払いで回復

したがって、**どんな順序でも途中で不足することはない**ため、選択肢②は「起こることがない」ケースです。

#### ポイント
- **最悪ケースの分析**：すべての可能な順序の中で、最も厳しい状況を考える
- **収支計算**：収入と支出のバランスを正確に計算
- **順序依存性**：同じメンバー構成でも、順序によって途中経過が変わる
